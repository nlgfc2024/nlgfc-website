# Incident Note: 504 Gateway Timeout on `/nlgfc-website/`

- Date: 2026-04-10
- Environment: Production
- Public URL: `http://41.77.13.188:8080/nlgfc-website/`
- Error: `504 Gateway Time-out` (`nginx/1.24.0`)

## Symptoms

- Page routes under `/nlgfc-website/*` timed out after ~60s.
- API routes such as `/api/contact`, `/api/vacancies`, and `/api/projects/menu` responded with `200`.

## Root Cause

- SSR page rendering depended on API calls that could block long enough for nginx upstream timeout.
- Multiple code paths defaulted API base to `http://localhost:8000` when runtime env vars were missing.
- Navbar SSR requests were not guarded with fast fail/fallback behavior.

## Changes Implemented

## 1) Runtime config default hardening

- File: `nuxt.config.ts`
- Change:
  - `baseUrl` fallback changed from `http://localhost:8000` to empty string `''`
  - `apiBase` fallback changed from `http://localhost:8000` to empty string `''`

## 2) API fetch timeout/fallback hardening

- Files:
  - `composables/useApiData.ts`
  - `composables/useDonorProjects.ts`
  - `composables/usePosts.ts`
  - `composables/usePartners.ts`
  - `components/ImageSlider.vue`
- Change:
  - Added bounded fetch behavior (`timeout`, `retry: 0`) for critical API requests.
  - Normalized API base resolution to `config.public.apiBase || config.public.baseUrl || ''`.

## 3) SSR navbar resilience

- File: `components/Navbar.vue`
- Change:
  - Added `fetchWithFallback()` helper with timeout and safe fallback payload.
  - Wrapped vacancies/procurement/project menu API calls with fallback behavior so SSR does not block route response.

## 4) Consistent API base usage in pages

- Files:
  - `pages/index.vue`
  - `pages/contact.vue`
  - `pages/about.vue`
  - `pages/news.vue`
  - `pages/resourceCenter.vue`
  - `pages/resourceCenter copy.vue`
  - `pages/localAuthorities/alldistricts.vue`
  - `composables/useLAReports.ts`
- Change:
  - Removed hardcoded localhost fallback and aligned all to runtime-safe base resolution.

## 5) Deployment workflow env propagation

- File: `.github/workflows/deploy-production.yml`
- Change:
  - Added deploy step env wiring for:
    - `NUXT_PUBLIC_BASE_URL`
    - `NUXT_PUBLIC_API_BASE`
    - `NUXT_PUBLIC_JOBS_PORTAL_BASE`
  - Exported these vars on the remote server before build/restart.

## Validation

- Local build result: `npm run build` passed after changes.
- Required production verification after deploy:
  - `curl -s -o /dev/null -w "%{http_code} %{time_total}\n" http://41.77.13.188:8080/nlgfc-website/`
  - Expect `200` and normal response time (not ~60s timeout).
